Context

What is Scanability?

The ability of a scanner to reliably read a code

Scanability measures whether a printed or screen-displayed barcode or QR code can be successfully read across different devices, lighting conditions, and scanning angles. Poor scanability leads to failed payments and frustrated users.

For Rakuten Pay, two code types are used at checkout — both were tested and failed ISO/IEC grade thresholds.

Testing Scope

Barcode Verifier vs POS Systems

Barcode Verifier (BV)

Need: 1 unit
  • → Measures print/display quality against ISO/IEC standards
  • → 1 device covers all 20 smartphones — one scan per phone per code type
  • → Gives objective grade (A–F) with detailed failure reasons

POS Systems

Need: 4–5+ units
  • → Each POS scanner model behaves differently — all must be tested separately
  • → Real-world acceptance test: does this POS read our code under store conditions?
  • → Each code type on each POS must be verified individually

Background

Why Barcode Verification Matters

  • → Low print quality causes misreads and payment transaction failures
  • → Quiet zone, contrast & modulation must meet ISO/IEC standards
  • → Required for compliance: ISO 9000, GS1 specifications
  • → Directly affects Rakuten Pay and convenience store payment barcodes
Products Presented

REA Barcode Verifiers

✦ Evaluated
Stationary · High-end · 2D

REA VeriCube

5MP CMOS camera unit for verifying both 1D and 2D barcodes including QR codes. Supports 4 interchangeable camera modules (8mm–25mm). Includes DPM support.

DPM (Direct Part Marking) — codes physically engraved or etched onto surfaces, e.g. laser-marked parts. Harder to scan, requires specialized verification.
Portable · Budget · 1D

REA CheckER

Battery-powered portable CCD scanner for 1D barcodes only. PC-free field use. Lower cost entry point for basic verification needs.

Shared Management Software

TransWin32

Network-connected software for all REA verifiers. Configurable pass/fail thresholds, live camera view, PDF report export. Compatible with CheckER and VeriCube.

Assessment

REA VeriCube — Pros & Cons

✓ Advantages

  • ✓ ISO/IEC compliant — internationally recognized grading (A–F)
  • ✓ Detailed PDF reports with full parameter breakdown
  • ✓ Network-connected — centralized data management via TransWin32

✗ Disadvantages

  • ✗ Slow detection speed — not suitable for real-time or high-throughput scanning environments
  • ✗ High noise sensitivity — detects excessive noise, may flag codes that real POS scanners accept without issue
